Оплата ChatGPT через Плати по миру

Выпуск виртуальной карты, пополнение через СБП и оплата подписки

ChatGPT и API оплачиваются отдельно. Для подписки ChatGPT используйте сайт ChatGPT и раздел Billing, а не billing в OpenAI Platform. Перед оплатой проверьте условия OpenAI и условия сервиса «Плати по миру».
1

Открыть бота и мини-приложение

Перейдите в Telegram-бот @platipomiru_bot, нажмите Старт, затем Открыть приложение. Откроется личный кабинет «Плати по миру».

2

Выпустить карту для подписок

В мини-приложении выберите карту Для подписок, нажмите Выпустить карту и оплатите выпуск через СБП. После оплаты в кабинете появятся реквизиты карты.

3

Пополнить карту с запасом

Пополните карту рублями через СБП. Баланс карты «Для подписок» отображается в EUR, а оплата ChatGPT может списаться с конвертацией и комиссией сервиса, поэтому лучше держать небольшой запас сверх стоимости подписки.

4

Открыть оплату ChatGPT

Перейдите на chatgpt.com, войдите в нужный аккаунт, нажмите на профиль и выберите Upgrade Plan. Выберите нужный тариф, например Plus или Pro.

5

Ввести реквизиты карты

На странице оплаты укажите номер карты, срок действия и CVV из мини-приложения. Если ChatGPT просит данные плательщика или billing address, используйте корректные данные и ориентируйтесь на подсказки ChatGPT и поддержку «Плати по миру».

6

Подтвердить платёж

Проверьте тариф, сумму и период списания. Если всё верно, подтвердите оплату. При запросе подтверждения операции откройте мини-приложение «Плати по миру» и подтвердите платёж там.

7

Проверить подписку и управление оплатой

После успешной оплаты в ChatGPT откройте Settings → Billing. Там можно проверить активный план, историю платежей и управление подпиской.

8

Если платёж не проходит

Проверьте баланс с учётом комиссии и конвертации, статус карты, правильность реквизитов и ограничения тарифа. Если отказ повторяется, обратитесь в поддержку «Плати по миру» через бот и приложите текст ошибки или скрин оплаты.

Установка OmniRoute и Claude Code

Полное руководство по настройке на Windows

Примечание: Для поиска ответов на вопросы можно использовать https://chat.deepseek.com/ - мощный инструмент для поиска информации и решения задач.

Установка зависимостей

Git

Способ 1 — через winget (рекомендуется):

winget install --id Git.Git -e --source winget

Способ 2 — через установщик:

Скачать с https://git-scm.com/install/windows

Проверка установки:

git --version

Node.js

Скачать с https://nodejs.org/en/download

Рекомендуемые версии:

Node.js 22.22.2

Оптимальная, самая стабильная для OmniRoute

Node.js 20.20.2

Если нужна более старая LTS

Node.js 24.14.1

Если нужна новейшая версия

Проверка установки:

node -v
npm -v

Установка Claude Code

Если Claude Code ещё не установлен, выполните команду:

npm install -g @anthropic-ai/claude-code

Проверка установки:

claude --version

Установка OmniRoute

npm install -g omniroute

Запуск OmniRoute

omniroute
После запуска: Панель управления доступна по адресу http://localhost:20128

Пароль по умолчанию:

  • CHANGEME
  • admin123
  • 123456

2

Авторизация

Использовать AWS Builder ID или Google/GitHub для входа

Выберите удобный способ входа

3

Доступные модели

  • kr/claude-sonnet-4.5
  • kr/claude-haiku-4.5

Модели готовы к использованию

Qwen Code

Бесплатный провайдер от Alibaba Cloud с доступом к специализированным кодовым моделям.

Qoder AI

Бесплатный провайдер с поддержкой различных AI моделей для программирования.

Gemini CLI

Инструмент командной строки для работы с моделями Gemini (устаревший, но все еще доступен).

Для большинства бесплатных провайдеров авторизация происходит через панель управления OmniRoute. После подключения они становятся доступны через единый API endpoint.

Все эти провайдеры входят в состав OmniRoute и не требуют дополнительных API ключей от вас.

Настройка Claude Code

Файл конфигурации

Файл settings.json находится в папке:

C:\Users\ВАШ_ПОЛЬЗОВАТЕЛЬ\.claude\

Папка создается автоматически при установке Claude Code.

Добавить блок "env"

Откройте файл C:\Users\ВАШ_ПОЛЬЗОВАТЕЛЬ\.claude\settings.json и добавьте:

{ "env": { "ANTHROPIC_API_KEY": "ВАШ КЛЮЧ ИЗ OmniRoute", "ANTHROPIC_BASE_URL": "ВАША ССЫЛКА ИЗ OmniRoute, ПО УМОЛЧАНИЮ http://localhost:20128/v1", "API_TIMEOUT_MS": "3000000", "ANTHROPIC_DEFAULT_HAIKU_MODEL": "kr/claude-haiku-4.5", "ANTHROPIC_DEFAULT_SONNET_MODEL": "kr/claude-sonnet-4.5", "ANTHROPIC_DEFAULT_OPUS_MODEL": "qw/qwen3-coder-plus" } }
В полях ANTHROPIC_DEFAULT_HAIKU_MODEL, ANTHROPIC_DEFAULT_SONNET_MODEL и ANTHROPIC_DEFAULT_OPUS_MODEL вы сами задаёте модели, которые вам нужны

Подтверждение API ключа при первом запуске

После настройки файла конфигурации при первом запуске Claude Code может появиться вопрос:

Do you want to use this API key?

Необходимо подтвердить использование ключа:

1. Yes

Это действие подтверждает использование API ключа из настроек и позволяет Claude Code корректно подключиться к OmniRoute.

Возможные проблемы и решения

OmniRoute не запускается

Решение 1: Проверить версию Node.js (рекомендуется 22.22.2)

node -v

Решение 2: Переустановить OmniRoute

npm uninstall -g omniroute && npm install -g omniroute

Решение 3: Запустить с явным указанием порта

omniroute --port 20128

Ошибка "Tools for Native Modules"

Решение:

1

Запустить PowerShell от администратора

2

Выполнить команду

npm install --global windows-build-tools
3

Дождаться полной установки

Может занять 5-10 минут

Порт 20128 занят

Решение 1: Проверить кто занимает порт

netstat -ano | findstr :20128

Решение 2: Запустить OmniRoute на другом порту

omniroute --port 20129

Решение 3: Обновить ANTHROPIC_BASE_URL в settings.json

"ANTHROPIC_BASE_URL": "http://localhost:20129/v1"

Claude Code не видит OmniRoute

Проверка 1: Убедиться что OmniRoute запущен и работает

Проверка 2: Проверить доступность в браузере

http://localhost:20128

Проверка 3: Проверить правильность API ключа в settings.json

Проверка 4: Выполнить диагностику в Claude Code

/doctor

Важные примечания

OmniRoute должен быть запущен перед использованием Claude Code
По умолчанию OmniRoute работает на порту 20128
Рекомендуемая версия Node.js: 22.22.2 (наиболее стабильная для работы с OmniRoute)
Все модели доступны через единый эндпоинт http://localhost:20128/v1
